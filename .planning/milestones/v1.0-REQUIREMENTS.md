# Requirements: Ship

**Defined:** 2025-02-01  
**Core Value:** Agent works autonomously in the background on real coding tasks while you do other things

## v1 Requirements

Requirements for initial release. Each maps to roadmap phases.

### Authentication

- [x] **AUTH-01**: User can sign in with GitHub OAuth
- [x] **AUTH-02**: User session persists across browser sessions
- [x] **AUTH-03**: LLM API keys configured via environment/config

### Sessions & Tasks

- [x] **SESS-01**: User can create, view, delete sessions
- [x] **SESS-02**: User can chat with agent in session
- [x] **SESS-03**: User can create tasks from chat messages
- [x] **SESS-04**: Session state persists in Durable Objects
- [x] **SESS-05**: User sees real-time status updates via WebSocket
- [x] **SESS-06**: Session can contain multiple tasks
- [x] **SESS-07**: Tasks process in FIFO order

### Sandbox & Execution

- [x] **SAND-01**: System provisions E2B sandbox per session
- [x] **SAND-02**: User can access code-server (VS Code) in sandbox
- [x] **SAND-03**: User can access terminal in sandbox
- [x] **SAND-04**: Agent executes multi-step tasks autonomously
- [x] **SAND-05**: Agent handles errors and recovers gracefully

### Git & Code

- [x] **GIT-01**: Agent can clone repos into sandbox
- [x] **GIT-02**: Agent can create branches
- [x] **GIT-03**: Agent can commit changes
- [x] **GIT-04**: Agent can push to remote
- [x] **GIT-05**: Agent can create GitHub PRs
- [x] **GIT-06**: User can view diffs in UI

### Integrations

- [x] **INTG-01**: System syncs issues from Linear (manual sync available)
- [x] **INTG-02**: Agent updates Linear issue status (conditional on explicit linking)
- [ ] **INTG-03**: Agent can create Linear issues (deferred - requires OpenCode SDK custom tool support)
- [x] **INTG-04**: System links GitHub PRs to Linear issues (conditional on explicit linking)
- [x] **INTG-05**: Vercel available as MCP in chat (MCP server structure ready, requires OpenCode SDK support)
- [x] **INTG-06**: User can enable/disable connectors globally

### Agent & AI

- [x] **AGNT-01**: OpenCode SDK powers agent runtime
- [x] **AGNT-02**: Agent has codebase context awareness
- [x] **AGNT-03**: User can select model (Claude, GPT, Gemini)
- [x] **AGNT-04**: User sees cost per task

### UI & Experience

- [x] **UI-01**: Chat interface matches Ramp Inspect design
- [x] **UI-02**: Session list with history
- [x] **UI-03**: Real-time status indicators
- [x] **UI-04**: Code viewer shows agent's changes live
- [x] **UI-05**: Terminal viewer shows agent output

## Milestone Summary

**Shipped:** 34 of 36 v1 requirements

**Adjusted:**
- INTG-01: Changed from automatic sync to manual sync (user-initiated only)
- INTG-02: Changed to conditional updates (only when Linear issue explicitly linked)
- INTG-04: Changed to conditional PR linking (only when Linear issue explicitly linked)

**Dropped:** None

**Deferred:**
- INTG-03: Linear issue creation from agent (requires OpenCode SDK custom tool support)
- INTG-05: Full Vercel MCP integration (MCP server structure ready, requires OpenCode SDK MCP support)

**Outcomes:**
- All authentication requirements validated
- All session management requirements validated
- All sandbox and execution requirements validated
- All Git workflow requirements validated
- All UI requirements validated
- Integration requirements mostly validated (2 deferred pending SDK support)
